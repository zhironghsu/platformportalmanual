Server-Sent Events (SSE)

Server-Sent Events (SSE) is a server push technology enabling a client to receive automatic updates from a server via HTTP connection. The Server-Sent Events EventSource API is standardized as part of HTML5 by the W3C.

Receiving events from the server

Our SSE APIs require token to access the data from the server. You can Login with OS account to get the token (token field from the response body), or using Secure and HttpOnly cookies (if available).

// Open a connection
const evtSource = new EventSource("/THE_SSE_API?token=<token>", { withCredentials: true } );

// Handle events
evtSource.addEventListener(<event type>, (event) => {
  const data = JSON.parse(event.data);
  // process your data
});

See Also

https://developer.mozilla.org/en-US/docs/Web/API/Server-sent_events

Invoking a REST API via Postman

Prerequisite

  1. Turn off SSL certificate verification

  2. Login with OS account to get token

  3. See Also

    POST /auth/login

Calling GET /platform/api/host/info

  1. Switch to Authorization Tab and set TYPE to Bearer Token

  2. Input the token from Prerequisite Step 2

  3. Send the Request

  4. See Also

    GET /platform/api/host/info

Calling PUT /platform/api/perif/wdt

  1. Switch to Authorization Tab and set TYPE to Bearer Token

  2. Input the token from Prerequisite Step 2

  3. Switch to Body Tab, select raw, JSON, and input the request body

    {
    "enabled": true
}

  4. Send the Request

  5. See Also

    PUT /platform/api/perif/wdt
