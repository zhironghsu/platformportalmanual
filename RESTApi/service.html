Service

(since rest-server 1.0.1)

GET /platform/api/service/list

Get the service list

  • Request Headers

    Log in with the OS or the "admin" account to get a JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Response 200 OK

    [
  {
      "name": "AppHubEdgeService",
      "displayName": "AppHub-Edge Service",
      "description": "",
      "status": "running"
  },
  {
      "name": "ALG",
      "displayName": "Application Layer Gateway Service",
      "description": "Provides support for 3rd party protocol plug-ins for Internet Connection Sharing",
      "status": "stopped"
  },
  ..
]

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 422 Unprocessable Entity

    The request has been applied but got errors.

    {
    "errors": {
        "Service": "<the detail error message>"
    }
}

GET /platform/api/service/:name

Get the service status by name.

  • Request Headers

    Log in with the OS or the "admin" account to get a JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Response 200 OK

    The available status will be "running", "stopped", "disabled", or "unknown".

    {
  "status": "running"
}

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 422 Unprocessable Entity

    The request has been applied but got errors.

    {
    "errors": {
        "Service": "<the detail error message>"
    }
}

PUT /platform/api/service/:name

Control the service status by name.

  • Request Headers

    Log in with the OS or the "admin" account to get a JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Request Body

    {
  "action": "start" | "stop" | "restart"
}

  • Response 204 No Content

    The requested operation is successful.

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 422 Unprocessable Entity

    The request has been applied but got errors.

    {
    "errors": {
        "Service": "<the detail error message>"
    }
}
