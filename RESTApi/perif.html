Peripheral

Peripheral

LED

GET /platform/api/perif/led

Get LED list with high-low value.

  • Request Headers

    Login with OS account to get JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Response 200 OK

    true -> logical high, false -> logical low

    [
  true,
  false,
  true,
  false,
  true,
  false,
  true,
  true
]

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 404 Not Found

    Not supported or no available data.

GET /platform/api/perif/led/:id

Get the specified LED value by index (id)

  • Request Headers

    Login with OS account to get JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Response 200 OK

    true -> logical high, false -> logical low

    true | false

  • Response 400 Bad Request

    {
    "errors": {
        "LED": "<the detail error message>"
    }
}

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 404 Not Found

    Not supported or no available data.

PUT /platform/api/perif/led/:id

Set value of the specified LED by index (id)

  • Request Headers

    Login with OS account to get JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Request Body

    true -> logical high, false -> logical low

    {
    "status": true | false
}

  • Response 200 OK

    <same as the request body>

  • Response 400 Bad Request

    Invalid Parameters Error

    {
    "errors": {
        "LED": "<the detail error message>"
    }
}

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 404 Not Found

    Not supported or no available data.

  • Response 422 Unprocessable Entity

    The request body has missing parameters.

    {
    "errors": {
        "validator": "<the detail validator error>"
    }
}

    The request has applied, but got errors.

    {
    "errors": {
        "LED": "<the detail error message>"
    }
}

Watchdog

GET /platform/api/perif/wdt

Get hardware watchdog information.

  • Request Headers

    Login with OS account to get JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Response 200 OK

    {
  "enabled": <true | false>,
  "type": "EC Watchdog",
  "timeoutInSeconds": 15,
  "timeoutMaxInSeconds": 255,
  "timeoutMinInSeconds": 10
}

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 404 Not Found

    Not supported or no available data.

  • Response 422 Unprocessable Entity

    The request has applied, but got errors.

    {
    "errors": {
        "WDT": "<the detail error message>"
    }
}

PUT /platform/api/perif/wdt

Set hardware watchdog enabled or disabled with provided timeout value (optional).

  • Request Headers

    Login with OS account to get JWT token (token field from the response body).

    Authorization: Bearer <token>

  • Request Body

    The seconds value range is [timeoutMinInSeconds, timeoutMaxInSeconds]

    {
  "enabled": <true | false>, // required
  "seconds": [timeoutMinInSeconds, timeoutMaxInSeconds], // optional
}

  • Response 200 OK

    {
  "enabled": <same as the request body>,
  "seconds": <same as the request body>
}

  • Response 401 Unauthorized

    {
    "Code": 401,
    "Message": "<the detail error message>"
}

  • Response 404 Not Found

    Not supported or no available data.

  • Response 422 Unprocessable Entity

    The request has applied, but got errors.

    {
    "errors": {
        "WDT": "<the detail error message>"
    }
}
