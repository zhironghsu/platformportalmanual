EdgeX to AWS IoT Core (Uplink)

EdgeX to AWS IoT Core (Uplink)

Create a Thing

  1. Login to AWS Management Console

  2. Search IoT Core in Find Services

  3. Select Manage > Things from the AWS IoT menu pane. Then Click Create button.

  4. The wizard displays, then click Create a single thing and follow the wizard to create a single Thing as a target for export data from EdgeX. For more information on using the wizard, refer to the AWS Documentation.

    4.1. Create a single thing

    4.2. Name your IoT thing

    4.3. Create certificate

    4.4. Download Certificate, Public key, and Private key. Then click Activate button to activate the certificate on AWS.

    Note: You need to download the certificate and keys for setting the EdgeX MQTTSecretSend pipeline.

  5. Click the Done button

Creating a Policy

  1. Select Secure > Policies from the AWS IoT menu pane

  2. Click Create button

  3. Enter a name for your policy.

  4. Set suitable permissions. If you are unsure of the permissions you require, add an asterisk () to the Action and Resource ARN text boxes, and select the *Allow checkbox, as illustrated below:

  5. Click the Create button

Attaching the Policy to the Thing

  1. Select Manage > Things from the AWS IoT menu pane

  2. Select the Thing you created

  3. Select Security from the Things menu pane

  4. Select the certificate you created

  5. Select the Actions drop-down menu

  6. Select Attach policy, as illustrated below:

  7. Select the policy you created

  8. Click the Attach button

Noting the Endpoint

  1. Select Interact from the Things menu pane

  2. Note the Rest API Endpoint

    We will use it as BrokerAddress in the MQTTSecretSend pipeline.

Set up EdgeX pipeline

Open EdgeX UI, navigate to EdgeX > Uplink, then Click ＋ to add a Transform.

  • Select MQTTSecretSend in the EXPORT tab. (EdgeX 1.x)

  • Or MQTTExport for EdgeX 2.x

Enter all the parameters.

  • BrokerAddress

    tls://<your AWS IoT Thing Rest API Endpoint>:8883

  • Topic: <your topic>
  • ClientId: <your clientId>
  • Qos: 0
  • AuthMode: clientcert

  • Key PEM Block

    <the content of the key you download from 'Create a Thing' step> The file name is in the following format: xxxxxxxxxx-private.pem.key

  • Cert. PEM Block

    <the content of the certificate you download from 'Create a Thing' step> The file name is in the following format: xxxxxxxxxx-certificate.pem.crt

  • SecretPath: mqtt

Click deoply button to apply the pipeline configuration.

You can use the AWS MQTT client to subscribe the topic to monitor the data from the EdgeX.
